# Contribuir

Gracias por colaborar en Seedance Video App.

## Flujo de trabajo

1. Crea una rama desde `master`.
2. Haz cambios pequenos y revisables.
3. Ejecuta las comprobaciones locales descritas en `README.md`.
4. Abre una pull request explicando el cambio y como se ha probado.

## Revision de PRs

El propietario `@mmc98gt` debe revisar y aceptar las pull requests antes de fusionarlas.

## Seguridad

- No subas `.env`, claves API ni videos privados.
- Los resultados en `outputs/` estan ignorados por Git.
- Si necesitas documentar un endpoint real de Seedance, evita incluir secretos o capturas con tokens.
